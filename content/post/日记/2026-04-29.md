## 自动获取数据

### 为什么使用影刀

相较于影刀，我更多关注了哪些原生支持cli和开源的方案，一方面，支持cli就可以通过agent来自动化获取数据；另一方面，开源方案通常来说本地部署是不需要钱的。
但是后来我在试着根据这个[视频](https://www.bilibili.com/video/BV1ooDyBmE6v/?spm_id_from=333.337.search-card.all.click)来通过claude code调用playwrightcli，但是效果非常差，经常莫名其妙要求人工验证，也不知道哪里出问题了，之后有时间我准备在codex上再试试。
影刀使用还是很方便的，全是拖动节点，手动捕获目标区域。整个就是模拟人类点击滚动等操作，很不容易被**封禁**。关于费用方面，我目前也没发现他有什么收费的点，好像是多人合作使用一个的时候要付费，等于免费😆

### 本地大模型调用

我原本是在使用fastgpt来处理这个数据的，因为fastgpt可以显式的制定大模型的提示词，而且之前那个净水客服我也配置好了相应的大模型供应商。但是后来发现搞来搞去，最后就一个节点😅，于是fastgpt就显得没什么很大的必要了。我最后决定使用openai库来调用大模型处理，~~话说这应该是我第一次通过代码来本地调用大模型~~。先看一下main函数：

```python
def main() -> int:

log("加载环境变量")

load_dotenv()
parser = argparse.ArgumentParser(description="Call DashScope to generate output.json")
parser.add_argument("--prompt", default="prompt.txt", help="prompt file path")
parser.add_argument("--heads", default="data/order_heads.csv", help="order heads csv path")
parser.add_argument("--full", default="data/order_full.txt", help="full order text path")
parser.add_argument("--out", default="data/output.json", help="output JSON path")
parser.add_argument("--model", default=os.getenv("DASHSCOPE_MODEL", DEFAULT_MODEL))
parser.add_argument("--base-url", default=os.getenv("DASHSCOPE_BASE_URL", DEFAULT_BASE_URL))
parser.add_argument("--api-key", default=os.getenv("DASHSCOPE_API_KEY"))
parser.add_argument(
"--no-json-mode",
action="store_true",
help="do not send response_format=json_object to the model API",
)
args = parser.parse_args()
if not args.api_key:
print("缺少 DASHSCOPE_API_KEY。请先创建 .env，或用 --api-key 传入。", file=sys.stderr)
return 2
log("开始生成 output.json")
result = call_model(
prompt_path=Path(args.prompt),
order_heads_path=Path(args.heads),
order_full_path=Path(args.full),
model=args.model,
base_url=args.base_url,
api_key=args.api_key,
use_json_mode=not args.no_json_mode,
)
out_path = Path(args.out)
log(f"写入结果文件: {out_path}")
out_path.write_text(json.dumps(result, ensure_ascii=False, indent=2), encoding="utf-8")
order_count = len(result.get("orders", [])) if isinstance(result, dict) else 0
log(f"已生成: {out_path}")
log(f"订单数量: {order_count}")
return 0
```

首先这个main函数很明显的一点就是使用了argparser来解析命令行，分别获取数据文件位置，大模型相关信息（base_url、model、dashborad、api，promot），然后调用call_model这个函数来调用大模型。
在call_model函数中，核心就这些

```python
log(f"初始化模型客户端: model={model}, base_url={base_url}")

client = OpenAI(api_key=api_key, base_url=base_url)

kwargs: dict[str, Any] = {

"model": model,

"messages": [

{

"role": "system",

"content": "你是订单文本结构化助手。只返回严格 JSON，不要返回 Markdown 或额外说明。",

},

{"role": "user", "content": user_message},

],

"temperature": 0,

}

if use_json_mode:

kwargs["response_format"] = {"type": "json_object"}



log("开始请求大模型，等待返回...")

start = time.perf_counter()

response = client.chat.completions.create(**kwargs)

elapsed = time.perf_counter() - start

log(f"模型已返回，耗时 {elapsed:.1f} 秒")
```

### 写入excel

要求写入的表格如下：
![](https://somnusblog.oss-cn-shanghai.aliyuncs.com/images/%E6%88%AA%E5%B1%8F2026-04-29%2017.04.23.png)
xlsx文件本质是一个**压缩包**，把不同的sheet压缩到一块。通过python的zipfile可以对xlsx文件进行解压并读取，写入完成后，再通过zipfile对其重新压缩。
对于文件写入，是通过`xml.etree.ElementTree`这个模块来获得的，展开后的xlsx，本质都是xml结构，即openxml协议。
通过手动查找、增加和修改 XML 标签（例如表示行的 `<row>` 和表示单元格的 `<c>` 以及表示值的 `<v>`）来实现对工作表内容的编辑。

通过这样一套逻辑，也可以成功的把数据写入到指定的行和列。
